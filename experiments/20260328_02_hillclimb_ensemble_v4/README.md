# 20260328_02_hillclimb_ensemble_v4

## 目的
v3のHill Climbingアルゴリズムを改良し、精度向上と計算コスト削減を検証する。

## アプローチ
v3と同一の13モデルに対し、以下の4点を変更：

1. **ランク正規化（rank01）**: OOF/テスト予測を読み込み直後にグローバルランクで[0,1]に正規化。モデル間のスケール差を吸収
2. **GPU並列評価（CuPy）**: 全重みグリッドを一括GPU評価。CPUループの逐次AUC計算を置換
3. **同一モデルの再選択許容**: `remaining.remove()`を廃止し、毎イテレーションで全モデルを候補に。最終重みは`groupby().sum()`で合算
4. **AUCの二段階計算**: 探索中はGPU近似AUC（タイ処理省略）、最終確認のみCPU厳密AUC

### 実行環境
- **Kaggle T4 GPU**（CuPy使用、ローカルはCUDA Toolkit未インストールのため不可）
- OOF/テスト予測はKaggle Datasetとしてアップロード（private）
- カーネルID: `nyhalcyon/hill-climbing-ensemble-v4`

### 考察
- **精度改善は微小**: rank正規化と再選択による改善は+0.000005。LBでも差なし（0.91712）
- **計算コスト8倍削減**: GPU並列評価が最大の成果。今後のモデル追加時に大きなメリット
- **再選択の効果**: ridge_xgb_reproduceとmlp_orig_refが再選択された。特にridge_xgb_reproduceはIter 5で負の重み（-0.11）で追加され、初期重みを微調整。再選択がなければ到達できない構成
- **rank正規化の効果**: モデル間の確率キャリブレーション差を吸収し、重み探索が安定化。ただしこのデータでは元々のキャリブレーションが近いため大きな差にはならず

### v4+ 選択モデルと重み（8/15）

| モデル | 重み | v4比較 |
|---|---|---|
| realmlp_orig_ref | +0.3963 | v4: +0.4036 |
| ridge_xgb_reproduce | +0.3150 | v4: +0.4528 (減少) |
| **ridge_lgbm_reproduce** | **+0.1896** | **NEW** |
| tabm_orig_ref | +0.0998 | v4: +0.1017 |
| xgb_optuna_20fold | +0.0725 | v4: +0.0949 |
| **ydf_orig_ref** | **+0.0482** | **NEW** |
| xgb_depth1 | -0.0649 | v4: -0.0458 |
| mlp_orig_ref | -0.0565 | v4: -0.0772 |

### v4+ 考察
- **Ridge→LGBMが重み0.19で3位に選択**: Ridge→XGBとは異なるLGBMの誤差パターンが多様性に貢献。Ridge→XGBの重みが0.45→0.32に減少し、ツリー系の多様性が向上
- **YDFが重み0.05で選択**: BEST_FIRST_GLOBAL + RANDOM categoricalによる独自の誤差パターン。単体AUC (0.917) は低いがアンサンブルで貢献
- **catboost_orig_refとxgb_baselineが脱落**: Ridge→LGBMとYDFが代わりに入った形
- **LBは変化なし (0.91712)**: OOF改善（+0.000015）がLBに反映されない。アンサンブル精度の天井に近い
