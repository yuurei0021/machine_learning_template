# 20260331_09_knn_baseline

## 目的
KNNによるベースライン構築。インスタンスベースの手法でツリー系・NN系と直交する誤差パターンを狙い、アンサンブル多様性を確保する。

## アプローチ
- **特徴量**: 生値のみ（15カテゴリカル + 4数値 = OHE後30特徴量）
- **前処理**: One-Hot Encoding (drop="first") + StandardScaler（fold毎にfit）
- **モデル**: KNeighborsClassifier (k=51, weights="distance", Euclidean)
- **CV**: 5-Fold Stratified CV
- **テスト予測**: 全訓練データでrefit → predict

## 結果

| Fold | AUC | LogLoss | Accuracy |
|------|------|---------|----------|
| 1 | 0.8994 | 0.3789 | 0.8480 |
| 2 | 0.9011 | 0.3788 | 0.8492 |
| 3 | 0.9002 | 0.3791 | 0.8480 |
| 4 | 0.9010 | 0.3800 | 0.8498 |
| 5 | 0.8990 | 0.3837 | 0.8485 |
| **OOF** | **0.9001** | **0.3801** | **0.8487** |

## 考察
- 単体AUC 0.900はGNB (0.887) より高いがツリー系 (0.916+) やNN系 (0.919) より大幅に低い
- アンサンブルでの貢献はGNN (0.915, w=+0.030) やTabTransformer (0.917, w=+0.058) のように、単体精度よりも誤差の直交性で決まる
- 距離ベースの手法はツリー/NN/線形と全く異なる帰納バイアスを持つため、Hill Climbingで正の重みを得る可能性あり
